- **Deployment challenges:** Firebase Cloud Functions required Blaze plan upgrade (Estevan handled immediately). Cloud Build service account needed IAM permissions (resolved via Google Cloud Console). The Firebase client SDK had intermittent connectivity failures in Cloud Functions — data would load on some cold starts but not others. Resolved by switching the data loader from the Firebase SDK to direct Firestore REST API calls, plus adding middleware to ensure data is loaded before any request is served. Multiple deploy iterations required.
- **Architecture adaptation:** The spec called for `@lobby-suite/content-db` package for Firestore access. Estevan directed using Firebase Admin SDK directly instead. In Cloud Functions, the Admin SDK couldn't cross-project read without IAM grants, so the final solution uses the public Firestore REST API — the most reliable approach for reading from a different project's open database.
- **Demo mode:** Estevan requested an auto-play demo mode for screen recording. Built as an Easter egg — tapping "Developed at 626 Labs" footer on the start screen triggers automated gameplay with paced answers and a deliberate miss on round 4 to showcase streak breaking.
